# 🚀 动态规划进阶指南：三大高阶模型解析

## 0、 背包问题专题 (Knapsack Series)

### 1. 0/1 背包：每种物品仅一件
* **状态转移**：`dp[j] = max(dp[j], dp[j - w[i]] + v[i])`
* **空间优化**：必须 **逆序** 遍历容量（从 `W` 到 `w[i]`）。
* **物理本质**：为了保证在计算 `dp[j]` 时，用到的 `dp[j - w[i]]` 是“上一行”的旧值，从而确保每件物品只被选一次。

### 2. 完全背包：每种物品无限件
* **状态转移**：`dp[j] = max(dp[j], dp[j - w[i]] + v[i])`
* **空间优化**：必须 **顺序** 遍历容量（从 `w[i]` 到 `W`）。
* **物理本质**：由于可以重复选取，我们需要利用“本行”已经更新过的值，即在选取了第 `i` 件物品后，依然可以在此基础上继续选取。

### 📊 背包对比总结
| 特性 | 0/1 背包 | 完全背包 |
| :--- | :--- | :--- |
| **物品限制** | 每样 1 件 | 每样无限 |
| **一维遍历顺序** | **从大到小 (逆序)** | **从小到大 (顺序)** |
| **核心目的** | 防止物品重复使用 | 允许物品重复使用 |

## 1. 🧬 子序列与子串模型 (双序列比对)
处理两个字符串或序列之间的匹配关系。核心在于二维备忘录 `dp[i][j]`，以及对“连续性”的物理定义。

### 核心辨析
* **子序列 (Subsequence)**：可以跳过字符（不连续），但必须保持相对顺序。
* **子串 (Substring)**：必须**绝对连续**。

### 状态转移方程对比
假设我们正在对比 `text1[i-1]` 和 `text2[j-1]`：

| 匹配情况 | 🎒 最长公共子序列 (LCS) | 📏 最长公共子串 |
| :--- | :--- | :--- |
| **字符相等** | `dp[i][j] = dp[i-1][j-1] + 1` | `dp[i][j] = dp[i-1][j-1] + 1` |
| **字符不相等** | `dp[i][j] = max(dp[i-1][j], dp[i][j-1])` <br>*(退一步继承历史)* | `dp[i][j] = 0` <br>*(连续性断裂，当场清零)* |

*注：子串问题的最终结果可能出现在中间，需要用一个变量 `max_len` 在遍历时实时记录打擂台。*

## 2. ⚙️ 状态机模型 (带限制条件的序列决策)
处理同一天内可能有多种复杂处境的问题（如：冷冻期、手续费、最多交易 K 次）。
**核心心法**：`dp` 数组记录的永远是**晚上 11:59 的账户快照**。把白天的“动作”和晚上的“状态”彻底分开。

### 经典三状态定义 (以包含冷冻期的股票为例)
* **状态 0 (📈 持有)**：今晚手里有一只股票。
  * `dp[i][0] = max(dp[i-1][0], dp[i-1][2] - prices[i])` (昨天就持有 vs 昨天自由身今天买入)
* **状态 1 (🧊 刚卖出)**：今天白天刚卖出，触发明天的冷冻惩罚。
  * `dp[i][1] = dp[i-1][0] + prices[i]` (昨天必须持有，今天卖出)
* **状态 2 (🕊️ 自由身)**：手里无股票，且明天可以自由买入。
  * `dp[i][2] = max(dp[i-1][2], dp[i-1][1])` (昨天自由今天继续空仓 vs 昨天刚卖出熬过了冷冻期)

## 3. 📏 区间模型 (合并与拆分)
处理相邻元素的合并或大区间的拆分问题。
**核心心法**：大区间依赖小区间。抛弃从左到右的线性思维，**按区间长度从小到大遍历**。

### 灵魂三层循环骨架
```cpp
// 1. 外层：枚举区间规模 len (从最小合并单位开始)
for (int len = 2; len <= n; ++len) {
    // 2. 中层：枚举区间起点 i (保证右边界 j 不越界)
    for (int i = 0; i <= n - len; ++i) {
        int j = i + len - 1; // 自动定位右边界
        
        // 3. 内层：枚举劈开区间的断点 k
        for (int k = i; k < j; ++k) {
            // 状态转移：左边历史 + 右边历史 + 最后一次大合并的消耗
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j] + cost(i, j));
        }
    }
}
