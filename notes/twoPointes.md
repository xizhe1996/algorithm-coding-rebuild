# 知识点卡片：双指针 (Two Pointers)

## 📌 核心特征
* **物理模型**：两个探子在数组/字符串上协同作战。主要分为两大阵营：
  1. **相向双指针 (左右夹击)**：探子分别在两头，向中间靠拢。主要利用数组的**有序性**或**单调性**来排除无效答案。
  2. **同向双指针 (快慢指针)**：探子都在同一头，一快一慢。常用于链表找环、数组原地去重、以及滑动窗口。
* **核心威力**：通过指针的规律性移动，跳过大量的无效枚举，通常能把 $O(N^2)$ 的时间复杂度降维到 $O(N)$。

## ⚔️ 经典例题
* **相向夹击**：LeetCode 167 两数之和 II - 输入有序数组、LeetCode 11 盛最多水的容器
* **同向快慢**：LeetCode 26 删除有序数组中的重复项、LeetCode 283 移动零
* **终极 Boss**：LeetCode 15 三数之和 (排序 + 循环定点 + 双指针夹击)

## 🛠️ SOP 编程模版 (相向双指针)

以“有序数组两数之和”为例：

```cpp
vector<int> twoSum(vector<int>& numbers, int target) {
    // 1. 🧱 初始化探子，分别驻扎在两端
    int left = 0;
    int right = numbers.size() - 1;

    // 2. 🛑 探子开始相向而行，相遇时停止 (避免重复使用同一个元素)
    while (left < right) {
        int sum = numbers[left] + numbers[right];
        
        if (sum == target) {
            // 🎯 命中目标，立刻返回 (根据题意处理下标)
            return {left + 1, right + 1}; 
        } else if (sum > target) {
            // 📉 当前结果太大，右探子向左走，寻找更小的值
            right--; 
        } else {
            // 📈 当前结果太小，左探子向右走，寻找更大的值
            left++; 
        }
    }
    
    // 3. 🚪 未找到时的默认返回
    return {}; 
}
```

### 🧠 避坑指南：
* **前提条件**：使用相向双指针夹击时，**数组必须是有序的**。如果原数组无序，需要先评估能否进行 $O(N \log N)$ 的排序，排序后是否会破坏题目要求的下标（如果要求返回原下标，就不能直接排序）。
* **越界安全**：始终保持 `left < right` 作为循环主条件，这样即使找不到目标，指针在中间相遇后也会安全退出，绝不会出现段错误。
