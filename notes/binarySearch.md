# 知识点卡片：二分查找 (Binary Search)

## 📌 核心特征
* **适用场景**：有序数组查找、寻找边界、以及**二分答案（将最优化问题转化为判定问题）**。
* **核心难点**：边界收缩逻辑（加一还是减一）、中点计算（向上还是向下取整）、死循环陷阱。

## ⚔️ 经典例题
* **基础查找**：LeetCode 704 二分查找
* **寻找边界**：LeetCode 34 在排序数组中查找元素的第一个和最后一个位置
* **二分答案**：LeetCode 875 爱吃香蕉的珂珂（将“最小速度”转化为“以此速度能否吃完”）

## 🛠️ SOP 与核心技巧

### 1. 两种核心写法区别
* **`while (left <= right)` (闭区间法 - 推荐)**：
  * **区间定义**：`[left, right]`，两端都包含。
  * **退出条件**：`left > right`，此时区间为空。
  * **收缩动作**：`left = mid + 1` 或 `right = mid - 1`。因为 `mid` 已经检查过，必须把它踢出下一次的闭区间。
* **`while (left < right)` (左闭右开/边界收敛法)**：
  * **区间定义**：寻找一个特定的收敛点。
  * **退出条件**：`left == right`，循环结束时 `left` 就是最终答案，不需要再判断。
  * **收缩动作**：通常是 `right = mid` 或 `left = mid + 1`。

### 2. 向上取整 vs 向下取整
* **向下取整（防溢出）**：`mid = left + (right - left) / 2`。默认写法，偏向左侧。
* **向上取整**：`mid = left + (right - left + 1) / 2`。偏向右侧。
  * **⚠️ 避坑**：如果你的收缩逻辑里有 `left = mid`，**必须**使用向上取整。否则当 `left` 和 `right` 只差 1 时，向下取整会导致 `mid == left`，区间无法缩小，陷入死循环。

### 3. 二分答案 (建模 SOP)
1. **确定单调性**：答案越大，越容易满足条件；答案越小，越不能满足（或者反之）。
2. **确定上下界**：粗略估计答案的最小值 `left` 和最大值 `right`。
3. **编写 `check(mid)` 函数**：判定在 `mid` 这个值下，是否满足题目条件。
