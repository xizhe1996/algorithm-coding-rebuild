# Day 1 - 哈希表入门：查找与判重

## 核心知识点
哈希表最常见的用途有三类：

### 1. 判断某个元素是否存在
常用：
- `unordered_set`
- `unordered_map.find()`

典型场景：
- 判重
- 判断是否出现过
- 判断某个值是否已经存在

### 2. 统计频次
常用：
- `unordered_map<Key, int>`

典型场景：
- 字符/数字出现次数统计
- 异位词
- 高频元素

### 3. 建立映射关系
常用：
- `unordered_map<Key, Value>`

典型场景：
- 值 -> 下标
- 字符 -> 位置
- 前缀和 -> 最早出现位置

---

## 题型识别信号
看到下面这些描述时，优先考虑哈希表：

- 是否存在
- 是否重复
- 出现过没有
- 频次统计
- 快速查找
- 一边遍历一边匹配

## 代表题

### LeetCode 1 - Two Sum
#### 思路
遍历数组时，检查 `target - nums[i]` 是否已经出现过。
如果出现过，说明找到了答案；否则把当前元素和下标存入哈希表。

#### 核心点
- 先查再插
- 哈希表存的是：值 -> 下标

#### 模板
```cpp
unordered_map<int, int> mp;
for (int i = 0; i < nums.size(); ++i) {
    int need = target - nums[i];
    auto it = mp.find(need);
    if (it != mp.end()) return {it->second, i};
    mp[nums[i]] = i;
}
